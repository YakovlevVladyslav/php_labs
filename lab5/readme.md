## Описание лабораторной работы
Целью данной работы является освоение принципов объектно-ориентированного программирования (ООП) в PHP. В ходе работы реализованы:

- Инкапсуляция данных в классах.

- Разделение ответственности между хранилищем (Repository), бизнес-логикой (Manager) и отображением (Renderer).

- Использование интерфейсов для обеспечения гибкости архитектуры (Dependency Inversion).

- Строгая типизация и документирование кода по стандарту PHPDoc.

### Задание 3. Бизнес-логика (TransactionManager)
Класс TransactionManager выполняет расчеты и сортировку. Он работает с интерфейсом, что делает его независимым от конкретной реализации хранилища.

```PHP
public function calculateTotalAmount(): float {
    $total = 0.0;
    foreach ($this->repository->getAllTransactions() as $transaction) {
        $total += (float) $transaction->getAmount();
    }
    return $total;
}
```

Контрольные вопросы
1. Зачем нужна строгая типизация в PHP и как она помогает при разработке?
Строгая типизация (strict_types=1) заставляет PHP проверять соответствие типов данных. Это помогает предотвратить логические ошибки, делает код предсказуемым и упрощает отладку, так как ошибки несоответствия типов обнаруживаются немедленно.

2. Что такое класс в ООП и какие основные компоненты класса вы знаете?
Класс — это шаблон (чертеж) для создания объектов. Основные компоненты: свойства (состояние), методы (поведение), конструктор (инициализация) и модификаторы доступа (public, private, protected).

3. Объясните, что такое полиморфизм и как он может быть реализован в PHP?
Полиморфизм — это возможность объектов разных классов иметь одинаковый интерфейс. В PHP он реализуется через интерфейсы или наследование, позволяя коду работать с абстракциями, а не с конкретными реализациями.

4. Что такое интерфейс в PHP и как он отличается от абстрактного класса?
Интерфейс — это контракт, который содержит только объявления методов без их реализации. В отличие от абстрактного класса, интерфейс не может содержать программный код и свойства, а один класс может реализовать множество интерфейсов.

5. Какие преимущества дает использование интерфейсов при проектировании архитектуры приложения?
Интерфейсы позволяют реализовать принцип инверсии зависимостей. В данной работе это позволило TransactionManager работать с любым типом хранилища (массив, БД), не меняя свою внутреннюю логику.
